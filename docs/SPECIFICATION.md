## 1. Онбординг (3 экрана)

При первом запуске приложения пользователь видит серию из трёх приветственных экранов.

**Онбординг 1** знакомит с основной функцией: на коробки наклеиваются QR-коды. Пользователь может нажать кнопку «Далее» для перехода ко второму слайду или кнопку «Пропустить», чтобы сразу перейти на страницу входа.

**Онбординг 2** рассказывает о возможности сканирования QR-кодов для поиска вещей. Пользователь снова может нажать «Далее» для перехода к третьему слайду или «Пропустить» для перехода к странице входа.

**Онбординг 3** объясняет, как отслеживать статус перевозки. Здесь кнопка называется «Начать» — при её нажатии пользователь переходит на страницу входа. Кнопка «Пропустить» также доступна и ведёт на страницу входа.

## 2. Регистрация и вход

### Страница входа

На странице входа пользователь видит два поля: для email и для пароля. Оба поля обязательны для заполнения. При вводе email проверяется его формат — он должен соответствовать стандартному виду example@email.com. Пароль не может быть пустым(если же поле остается незаполненным, а кнопка "войти" нажата приходит уведомление об ошибке), пароль должен быть длинее 6 символов.

Основная кнопка «Войти» запускает проверку введённых данных. Если email и пароль верны, пользователь попадает на главную страницу — дашборд. Если же данные введены неверно, приложение уведомляет об ошибке.

Под формой есть дополнительные элементы. Кнопка «Продолжить с email» перенаправляет на страницу с авторизацией почты. Ссылка «Забыли пароль?» ведёт на страницу восстановления пароля. Ссылка «Нет аккаунта?» открывает страницу регистрации.

### Страница регистрации

На странице регистрации пользователь заполняет четыре поля: имя, email, пароль и подтверждение пароля. Все поля обязательны для заполнения.

Имя должно содержать минимум два символа(нужно ввести фамилию и имя). Email проверяется на корректность формата. Пароль должен быть не короче шести символов. Поле подтверждения пароля должно точно совпадать с введённым паролем — в противном случае система покажет ошибку.

После успешного заполнения всех полей и нажатия кнопки «Зарегистрироваться» пользователь автоматически попадает на дашборд.

Внизу страницы есть ссылка «Уже есть аккаунт?» — она возвращает пользователя на страницу входа.

## 3. Восстановление пароля

Процесс восстановления пароля состоит из трёх последовательных шагов. На любом этапе пользователь может вернуться к странице входа по ссылке «Вернуться ко входу».

**Шаг 1 — ввод email.** Пользователь вводит свой email. Система проверяет, существует ли такой email в базе данных. Если email найден, на него отправляется шестизначный код подтверждения, и пользователь переходит ко второму шагу. Если email не найден, система показывает ошибку.

**Шаг 2 — ввод кода подтверждения.** Пользователь вводит полученный по почте код. Код действителен в течение 15 минут. Если код верный, пользователь переходит к третьему шагу. Если код неверный, система сообщает об ошибке. Также предусмотрена кнопка «Отправить повторно» — после её нажатия новый код отправляется на email, но кнопка становится недоступной на 60 секунд для предотвращения частых запросов.

**Шаг 3 — установка нового пароля.** Пользователь придумывает новый пароль (минимум шесть символов) и подтверждает его, вводя повторно. Если пароли совпадают и проходят проверку длины, система сохраняет новый пароль и перенаправляет пользователя на страницу входа. Если пароли не совпадают или пароль слишком короткий, показывается соответствующая ошибка.

## 4. Дашборд (главный экран)

Дашборд — это главная страница, которую пользователь видит после входа в приложение. В верхней части экрана расположено приветствие с именем пользователя, а рядом отображается текущий переезд с адресами отправления и назначения(эти кнопки не кликабельны, служат только для информации)

Чуть ниже находится карточка текущего переезда с полными адресами(не кликабельна)

Статистика представлена четырьмя карточками: общее количество коробок, количество упакованных коробок, количество коробок в пути и количество доставленных коробок. Эти данные обновляются автоматически при изменении статусов коробок.

Ниже статистики расположен блок «Мои комнаты». Здесь пользователь видит список всех созданных комнат. Каждая комната отображается в виде аккордеона: видно название комнаты и количество коробок в ней. При нажатии на комнату она раскрывается, показывая список коробок, находящихся в этой комнате. Для каждой коробки указывается её номер, краткое содержимое и статус с цветным индикатором.

В нижней части экрана находится блок «Недавние коробки». Здесь показываются четыре последние созданные коробки. Для каждой отображается её номер и название комнаты. При нажатии на стрелку «Все коробки» пользователь переходит на полную страницу списка коробок,видит все созданные коробки в этом переезде.

В правом верхнем углу экрана находится иконка профиля, по нажатию на которую пользователь переходит в свой профиль, на вкладку "Профиль".

## 5. Страница «Мои коробки»

Эта страница открывается из бокового меню. В правом верхнем углу находится кнопка «Новая коробка» для создания новой коробки.

Сразу под заголовком расположена поисковая строка. Пользователь может искать коробки по содержимому или по номеру коробки. Поиск работает в реальном времени: при каждом вводе символа список коробок фильтруется. Если по запросу ничего не найдено, показывается сообщение «По вашему запросу ничего не найдено».

Ниже поиска находятся фильтры в виде кнопок. Доступны следующие фильтры: «Все», «Кухня», «Спальня», «Гостиная», «В пути», «Доставлено» (можно добавить другие кнопки). Фильтры можно комбинировать между собой. Например, выбрав фильтр «Кухня» и «В пути», пользователь увидит только те коробки, которые находятся на кухне и имеют статус «В пути».

Сами коробки отображаются в виде карточек. Каждая карточка содержит номер коробки, название комнаты, краткое содержимое (первые несколько слов,ставим оганичение на символы 20-23 символа) и статус с цветным индикатором. Если содержимое коробки не помещается в одну строку, при нажатии на карточку оно раскрывается полностью.

Если у пользователя ещё нет ни одной коробки, страница показывает сообщение «Нет коробок. Создайте первую коробку».

## 6. Страница «Создание коробки»

На этой странице пользователь создаёт новую коробку.

Поле «Комната» представляет собой выпадающий список с предустановленными вариантами: Тамбур, Холл, Кухня-гостиная, Кухня, Гостиная, Спальня, Кладовая, Санузел, Балкон, Библиотека, Ванная комната, Кабинет, Подвал, Гардероб. В конце списка есть пустое поле(кликабельное), в которое пользователь может ввести своё собственное название комнаты, если предложенных вариантов недостаточно.

Поле «Что внутри» представляет собой многострочное текстовое поле. Пользователь подробно описывает содержимое коробки, например «Кастрюли, тарелки, ножи, разделочная доска». Это поле обязательно для заполнения.

Блок «Фото содержимого» позволяет пользователю добавить фотографию того, что находится в коробке. Поддерживаются форматы PNG и JPG, максимальный размер файла — 5 МБ. Добавление фото не является обязательным, но рекомендуется для удобства распаковки.

Чекбокс «Хрупкое» позволяет отметить, что в коробке находятся хрупкие предметы. При отображении коробки в списке рядом с ней будет появляться соответствующий значок.

Кнопка «Сохранить и получить QR-код» запускает процесс сохранения. Система проверяет, что поле содержимого не пустое и комната выбрана. Если всё заполнено верно, показывается индикатор загрузки в поле справа от формы заполнения (где должен быть QR-код), затем генерируется QR-код, и пользователь переходит на страницу «QR-код». Если поля не заполнены, показываются соответствующие сообщения об ошибках.

## 7. «QR-код»

Эта страница появляется после успешного сохранения коробки. На экране отображается сгенерированный QR-код. Под QR-кодом указан уникальный идентификатор коробки — его можно записать вручную на коробке, если нет возможности распечатать QR-код (альтернатива для десктопной версии).

Пользователю доступны три кнопки. Кнопка «Распечатать» открывает диалог печати, позволяя распечатать QR-код на  бумаге. Кнопка «Добавить ещё коробку» перенаправляет на страницу создания коробки для быстрого добавления следующей. Кнопка «Готово» возвращает пользователя на страницу «Мои коробки», где только что созданная коробка уже отображается в списке(возможно после обновления страницы).

## 8. Страница «Логистика»

На этой странице пользователь управляет статусами коробок и может ознакомиться с предложениями перевозчиков.

В верхней части находится таблица со всеми коробками пользователя. Каждая строка содержит чекбокс для выбора коробки, номер коробки, название комнаты, краткое содержимое (с ограничением по длине для экономии места примерно 23 символа) и текущий статус с цветным индикатором. Статусы могут быть: Упакована, У водителя, В пути, Доставлена, Распакована.

Над таблицей расположен чекбокс «Выбрать все» — он позволяет выделить все коробки в таблице одним нажатием.

Для изменения статуса пользователь должен выбрать хотя бы одну коробку и нажать кнопку «Изменить статус». Если коробки не выбраны, система покажет ошибку «Коробка не выбрана». При выборе одной или нескольких коробок открывается модальное окно, в котором пользователь выбирает новый статус из предложенных вариантов (Упакована,У водителя, В пути, Доставлена, Распакована). Если пользователь выбирает статус ниже предыдущего, то система выводит ошибку. После подтверждения статусы выбранных коробок обновляются.

Кнопка «Обновить список» позволяет принудительно перезагрузить таблицу, чтобы увидеть актуальные статусы.

В нижней части страницы находится блок «Перевозчики». Здесь отображаются карточки с предложениями транспортных компаний. Каждая карточка содержит название перевозчика, рейтинг, примерную стоимость и контактную информацию. При нажатии на карточку пользователь переходит на официальный сайт перевозчика в новой вкладке.

Если у пользователя ещё нет ни одной коробки, таблица остаётся пустой, и отображается сообщение «Нет созданных коробок».

## 9. Страница «Сканер»

На этой странице пользователь может найти содержимое коробки, отсканировав QR-код.

При загрузке страницы система запрашивает доступ к камере устройства. После получения разрешения на экране отображается изображение с камеры, а также рамка для наведения на QR-код.

Система предлагает окрыть камеру телефона, если это ПК, то это окно заблокировано, систп ма ссылается на окно "Ввести номер вручную". При наведении камеры на QR-код система автоматически распознаёт его. Если распознавание успешно, пользователь видит окно с результатом, где указаны номер коробки, комната, полное содержимое, статус и отметка о хрупкости (если она была установлена). Если QR-код не распознаётся, показывается сообщение «QR не распознан, попробуйте снова или введите номер вручную».

Вместо сканирования пользователь может воспользоваться кнопкой «Ввести номер вручную». При её нажатии появляется поле для ввода идентификатора коробки. Если введённый идентификатор найден, отображается то же окно с результатом. Если не найден, показывается ошибка «Коробка не найдена. Проверьте номер».

В окне результата есть две кнопки. Кнопка «Отметить распакованной» меняет статус коробки на «Распакована» и перенаправляет пользователя на дашборд. Кнопка «Назад» закрывает окно результата, возвращая пользователя к камере для дальнейшего сканирования.

## 10. Страница «Профиль»

На странице профиля пользователь управляет своими данными и настройками.

В верхней части отображается имя пользователя и его email. Ниже находится блок «Контактная информация», где в виде таблицы показаны имя, email и номер телефона. Рядом с каждым полем есть кнопка «Изменить», которая позволяет пользователю обновить соответствующую информацию.

Блок «Уведомления» содержит настройки оповещений. Переключатель «Email уведомления» доступен всегда — он включает или отключает получение писем о смене статуса коробок. Переключатель «SMS уведомления» становится доступным только после подтверждения номера телефона.

Рядом с настройкой SMS-уведомлений находится статус подтверждения номера и кнопка «Подтвердить». При нажатии на неё открывается окно, где пользователь вводит свой номер телефона. На указанный номер отправляется SMS с кодом подтверждения. После ввода правильного кода номер считается подтверждённым, и SMS-уведомления становятся доступны для включения.

Блок «Мои переезды» содержит информацию об активном переезде и завершённых. Рядом с активным переездом есть кнопка «Подробнее» — при нажатии пользователь попадает на дашборд, где может работать с текущим переездом. Рядом с завершёнными переездами есть кнопка «Архив» — она ведёт на страницу архива переездов.

В самом низу страницы находится кнопка «Выйти из аккаунта». При её нажатии появляется модальное окно с вопросом: «Вы уверены, что хотите выйти?». Если пользователь подтверждает выход, он перенаправляется на первый экран онбординга. Если нажимает «Отмена», окно закрывается, и пользователь остаётся в профиле.

## 11. Страница «Архив переездов»

На этой странице пользователь видит все свои завершённые переезды.

В верхней части находится поисковая строка. Пользователь может искать переезды по названию или по городу назначения. Поиск работает в реальном времени: при вводе символов список переездов фильтруется. Если ничего не найдено, показывается сообщение «Переезды не найдены».

Каждый завершённый переезд представлен в виде отдельной карточки. На карточке указаны: название переезда, дата завершения, сколько месяцев или лет прошло с момента завершения, маршрут (адрес отправления и адрес назначения), общее количество коробок в этом переезде. В правой части карточки находится кнопка «Клонировать».

Если у пользователя ещё нет завершённых переездов, отображается сообщение «Нет завершённых переездов» и кнопка «К переездам», которая возвращает пользователя на дашборд.

---

## 12. Функция «Клонировать переезд»

Клонирование позволяет пользователю быстро создать новый переезд на основе завершённого, чтобы не создавать комнаты заново.

При нажатии на кнопку «Клонировать» открывается модальное окно с подтверждением. В окне объясняется, что будет скопировано, а что нет: копируются название переезда, адреса и все комнаты (названия и порядок). Коробки и их содержимое не копируются — пользователь будет паховать вещи заново в новом переезде.

Если пользователь подтверждает клонирование, система создаёт новый переезд. Новый переезд получает те же адреса и те же комнаты, что и исходный. После успешного создания показывается уведомление «Переезд «Название» склонирован!», и пользователь перенаправляется на дашборд, где может начать работу с новым переездом.

Если пользователь нажимает «Отмена» в модальном окне, клонирование отменяется, и он остаётся на странице архива.

## 13. Структура бокового меню

Боковое меню доступно на всех страницах после входа в приложение. Оно содержит следующие пункты:

- **Дашборд** — главная страница со статистикой и недавними коробками.
- **Мои коробки** — полный список всех коробок с возможностью поиска и фильтрации.
- **Логистика** — управление статусами коробок и справочник перевозчиков.
- **Сканер** — страница для сканирования QR-кодов и поиска содержимого коробок.
- **Профиль** — настройки пользователя, контактная информация и архив переездов.

В нижней части меню отображаются последние комнаты, которые пользователь создавал. При нажатии на название комнаты пользователь переходит на страницу «Мои коробки» с автоматически применённым фильтром по этой комнате.

## 14. Примечания для разработчиков

В процессе разработки необходимо учитывать следующие технические детали:

При переходе со страницы входа на регистрацию через кнопку «Продолжить с email» поле email должно быть предзаполнено тем значением, которое пользователь уже ввёл.

На странице восстановления пароля на каждом из трёх шагов должна быть возможность вернуться ко входу.

На дашборде при нажатии на стрелку «Все» в блоке «Мои комнаты» должен открываться полный список всех комнат. Аналогично, при нажатии на стрелку «Все» в блоке «Недавние коробки» должен открываться полный список всех коробок.

На странице «Мои коробки» поиск должен работать в реальном времени, без необходимости нажимать дополнительную кнопку. Фильтры могут комбинироваться между собой.

При нажатии на карточку коробки на странице «Мои коробки» полное содержимое должно раскрываться, если текст не помещается в одну строку.

В поле выбора комнаты при создании коробки должна быть возможность ввести своё название с помощью кнопки «+».

На странице «Логистика» при массовом изменении статуса необходимо проверять допустимость перехода для каждой выбранной коробки. Если у коробок разные статусы, система должна предлагать только те статусы, которые доступны для всех выбранных коробок.

На странице «Сканер» камера должна активироваться автоматически при загрузке страницы, с предварительным запросом разрешения.

На странице «Профиль» SMS-уведомления должны быть доступны только после того, как пользователь подтвердит свой номер телефона через код из SMS.

При клонировании переезда копируются только название, адреса и комнаты. Коробки и их содержимое не копируются.

После подтверждения выхода из аккаунта пользователь должен попадать на первый экран онбординга, а не на страницу входа.

## 15. Завершение переезда

### 15.1. Общая логика

Переезд завершается по **комбинированному принципу**: система отслеживает статус коробок и предлагает пользователю завершить переезд, но окончательное решение остаётся за пользователем.

### 15.2. Условие для предложения завершить переезд

Система начинает предлагать завершить переезд, когда **все коробки переезда имеют статус «Распакована»**.

При этом:
- На дашборде в карточке текущего переезда появляется кнопка-предложение «Завершить переезд»
- На странице профиля в блоке «Мои переезды» рядом с активным переездом появляется аналогичная кнопка
- Пользователь получает push-уведомление (в приложении) и email с предложением завершить переезд

**Важно:** переезд НЕ завершается автоматически. Система только предлагает, пользователь принимает решение.



### 15.3. Варианты действий пользователя

**Вариант 1: Пользователь нажимает «Да, завершить»**

Система выполняет следующие действия:
- Статус переезда меняется с «активный» на «завершён»
- Записывается дата завершения
- Переезд перемещается в архив
- Статистика переезда (количество коробок, маршрут, даты) сохраняется и становится доступной в разделе «Мои переезды» → «Архив»
- Пользователь получает финальное подтверждение

**Финальное уведомление:**

**Вариант 2: Пользователь нажимает «Нет, позже»**

Уведомление закрывается. Переезд остаётся в активном статусе. Система отправляет повторные напоминания о завершении этого переезда при каждом входе на страницу дашборд.


### 15.5. Что происходит после завершения

После того как пользователь подтвердил завершение:

1. Переезд больше не отображается как активный на дашборде
2. Переезд появляется в списке архива в разделе «Мои переезды»
3. Статистика переезда (количество коробок, маршрут, дата завершения) доступна для просмотра
4. Пользователь может клонировать этот переезд для быстрого создания нового
5. Все данные о коробках и комнатах сохраняются в истории

### 15.6. Если пользователь не завершает переезд

Если пользователь нажимает «Нет, позже» в уведомлении или просто игнорирует его:

- Переезд остаётся в активном статусе
- Все коробки остаются в статусе «Распакована»
- Повторные уведомления о завершении этого переезда появляются при каждом входе на дашборд


### 15.7. Блокирующие условия

Переезд **не может быть завершён**, если хотя бы одна коробка имеет статус отличный от «Распакована». В таком случае уведомление не отправляется.
